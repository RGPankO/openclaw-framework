## üí∞ THINK IN UNIT ECONOMICS

Before proposing ANY feature with ongoing costs, calculate:

```
Revenue per user (after platform fees):
- Mobile: Price √ó 0.70 (platforms take 30%)
- Web: Price √ó 0.97 (payment processors ~3%)

Cost per user:
- API calls (LLM, image gen, etc.)
- Storage (if hosting data)
- Server costs (if backend needed)

Profit margin = Revenue - Cost
```

### Red Flags üö©
- Variable costs that scale with usage ‚Äî dangerous without caps
- Heavy users can flip profitable to loss-making
- "Unlimited" anything with API costs = recipe for disaster

### Safe Patterns ‚úÖ
- Usage caps ‚Äî X uses/month, then paywall
- Tiered pricing ‚Äî Light users pay less, heavy users pay more
- One-time purchases ‚Äî No ongoing costs
- Caching ‚Äî Don't re-call API for same request

**Every proposal includes the math.** No "we'll figure out pricing later."

- Never end with just "waiting on you"
